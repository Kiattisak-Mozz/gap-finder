# Product

## Register

product

## Users

Gap Finder is for an independent frontend developer and AI builder looking for practical passive-income opportunities across Thailand, Southeast Asia, and global niches. The user is likely scanning opportunities between build sessions, comparing budget, time to ROI, competition, and feasibility before deciding what to build next.

## Product Purpose

Gap Finder is an AI opportunity dashboard for discovering, prioritizing, and later validating business gaps. Success means the dashboard helps the user quickly move from a list of possible ideas to a confident BUILD / PARK / KILL decision, starting with hardcoded Phase 1 data and growing into live scanning agents, skeptic analysis, and persistent project tracking.

## Brand Personality

Practical, energetic, builder-minded. The product should feel like a focused command center for spotting market gaps, not a generic SaaS template or a decorative analytics demo. Thai and English content can coexist because the opportunity space is intentionally cross-market.

## Anti-references

Avoid generic finance-dashboard decoration, passive-income hype, and vague AI buzzword copy. Avoid UI patterns that make the product feel like a Dribbble mockup before it feels like a working decision tool: excessive floating objects, decorative motion, over-rounded nested cards, soft shadows everywhere, and metrics that do not explain what action to take.

## Design Principles

1. Make the next decision obvious: every metric should help the user compare, prioritize, or act.
2. Prefer earned density: this is a work surface, so compact information is welcome when hierarchy stays clear.
3. Show confidence, not hype: frame passive-income opportunities with budget, effort, risk, competition, and time.
4. Keep Thai-market context visible: localization is a strategic advantage, not decorative flavor.
5. Motion must clarify state or focus: use GSAP for feedback and transitions, not page-load choreography.

## Accessibility & Inclusion

Target WCAG AA contrast and keyboard accessibility for all primary actions. Support reduced-motion preferences because the interface uses GSAP animation. Do not rely on color alone for status, ROI, difficulty, or readiness.
